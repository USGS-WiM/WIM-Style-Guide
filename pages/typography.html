Typography

The fonts and styles below should be used for all comminications, websites, and literature to ensure a consistent look and feel for everything both offline and online. All fonts are included in the wim.css file.

Font Families - Web

Merriweather should be used for all header and sub header text, and Open Sans should be used for all other copy (body, quotes etc). Using heavier font weights for headers and to highlight important messages.

Paragraphs

Automatic styling for <p> tags, <pre> tags, <code> tags, and <blockquote> tags.


<p> - Paragraph tag

This is an example of an automatically-styled paragraph tag. Stilton gouda cauliflower cheese. Cheese strings goat chalk and cheese boursin blue castello dolcelatte the big cheese swiss. Halloumi paneer st. agur blue cheese cheeseburger lancashire fromage frais cream cheese lancashire. Jarlsberg fromage frais airedale


<pre> - Pre tag

Text in a <pre> element is displayed in a fixed-width font, and it preserves both spaces and line breaks.

This is an example of an automatically-styled pre tag. Stilton gouda cauliflower cheese. Cheese strings goat chalk and cheese boursin blue castello dolcelatte the big cheese swiss. Halloumi paneer st. agur blue cheese cheeseburger lancashire fromage frais cream cheese lancashire. Jarlsberg fromage frais airedale

<code> - Code tag

The <code> tag can be used many places - normally within a paragraph element - to display a short block of computer code. It's not meant to be used for a code snippet. Pre would be better for full snippets. For example, you might use it when talking about which variable to use in documentation:

Example:
When accessing the total number of apples in a box, use $scope.applesInBox_total;, but when looking for only green apples in the box, use $scope.applesInBox_green;.


<blockquote> - Quote tag

The <blockquote> tag should be used for quotes from outside sources.

This is a quote from an outside source.
- Mitch Samuels

Example Code: 

<blockquote>
    This is a quote from an outside source
    <br>
    - Mitch Samuels
<blockquote>
